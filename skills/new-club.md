# new-club — 新クラブ展開手順（質問応答フロー）

ユーザーが「新しいチームのホームページを作りたい」「クラブのサイトを作って」などと言ったとき、この手順に従う。

---

## フロー全体像

```
Step 1: 受付
Step 2: 基本情報ヒアリング（必須項目）
Step 3: プラン＆デザイン選択
Step 4: 任意項目ヒアリング
Step 5: 確認→スクリプト実行→URL報告
```

---

## Step 1: 受付

最初に一言だけ：

> 「では、いくつか質問させてください。答えていただくだけで完成します。」

すぐ Step 1.5 の質問へ続ける。

---

## Step 1.5: 参考サイトURL確認

受付の直後にこの一言を添える：

```
参考にしたいサイトはありますか？
あればURLを教えてください（なければスキップでOKです）。
```

### URLが来た場合 → WebFetch で解析する

WebFetch でそのサイトを取得し、以下を自動抽出する：

| 抽出する情報 | 使い道 |
|------------|--------|
| ヘッダー背景色・ボタン色 | mainColor に自動反映 |
| アクセント・リンク色 | subColor に自動反映 |
| レイアウト傾向（ヒーロー大 / カード型 / テキスト重視） | デザインA/B/Cを自動推薦 |
| セクション構成（ニュース有？試合結果有？選手紹介有？） | featuresフラグを自動設定 |

解析後、抽出したカラーをユーザーに提示して確認を取る：

```
参考サイトから以下のカラーを取得しました。

メインカラー：#1a3a8f（ネイビー）
サブカラー：#e8c000（ゴールド）
デザイン推薦：B（躍動感・大きなヒーロー画像構成のため）

このままでよいですか？（変更したい場合は色名か16進数で教えてください）
```

**達成できること・できないことの目安：**
- 色の自動抽出：ほぼ完全対応
- デザイン方向（A/B/C）の自動推薦：参考サイトの雰囲気から最適を選ぶ
- セクション構成の最適化：featuresフラグで制御
- ピクセル完全一致：不可（固定テンプレートのため）

### URLが来なかった場合 → スキップ

そのまま Step 2 へ進む。

---

## Step 2: 基本情報ヒアリング

以下を **1つのメッセージ** にまとめて聞く。

```
① クラブの正式名称は？（例：FC グランデ）
② 略称を英字で教えてください（例：GRANDE）
③ メインカラーの16進数コードは？（例：#1a5c2d）
   ※わからない場合はユニフォームの色を教えてください（「緑」「青と白」など）
④ サブカラーは？（例：#ffffff）
   ※わからない場合は「白」「金」「黒」などでもOKです
```

**カラーが「わからない」「色名しかわからない」場合の対処：**
ユニフォームの色から近い16進数をClaudeが提案し、「これでよいですか？」と確認してから進む。
（例：「緑と白」→ mainColor: `#2d7a2d`, subColor: `#ffffff` を提案）

---

## Step 3: プラン＆デザイン選択

### プラン選択

回答が返ってきたらすぐ続けて聞く（技術用語は使わない）：

```
ページ数・機能量でプランが3つあります。

【ライト】
  ・トップ・チーム紹介・お問い合わせなど最小構成
  ・小規模クラブや立ち上げ初期に最適

【スタンダード】
  ・ニュース・試合結果・選手紹介・スクール情報なども搭載
  ・一般的なサッカークラブに最もよく合うプラン ← おすすめ

【プレミアム】
  ・スポンサー枠・物販・会員向けページなど全機能
  ・組織規模が大きいクラブ向け

どれにしますか？（迷ったらスタンダードがおすすめです）
```

### デザインスタイル選択

プラン選択後、同じメッセージまたは次のメッセージで続ける：

```
デザインのスタイルも3種類あります。

【A｜落ち着き・信頼感】
  余白を活かした読みやすいレイアウト。保護者に好まれるスタイル

【B｜躍動感・スポーツらしさ】
  大きな写真とダイナミックな構成。サッカーの迫力が伝わるデザイン

【C｜ミニマル・情報整理重視】
  カード型でスッキリ整理されたデザイン。更新頻度が高いクラブ向け

A・B・Cどれにしますか？
```

### プラン×デザイン → テンプレート対応表

| プラン＼デザイン | A | B | C |
|---------------|---|---|---|
| ライト | `light-01` | `light-02` | `light-03` |
| スタンダード | `standard-01` | `standard-02` | `standard-03` |
| プレミアム | `premium-01` | `premium-02` | `premium-03` |

---

## Step 4: 任意項目ヒアリング

以下を **1つのメッセージ** にまとめて確認（後から変更できると伝える）：

```
以下は任意です。後からでも変更できます。

⑤ キャッチコピーは？（例：「情熱を、グラウンドへ。」）
   ※なければ「{クラブ名}とともに、前へ。」にします
⑥ InstagramのアカウントがあればそのページのURL
⑦ LINE公式アカウントがあればそのURL
⑧ 住所・電話番号・メールアドレス（お問い合わせページに掲載するもの）
⑨ 設立年
```

---

## Step 5: 確認→実行→URL報告

### 確認メッセージ

収集した情報を整理してユーザーに提示：

```
以下の内容でホームページを作成します。

クラブ名：{name}
プラン：{プラン名}（デザイン{A/B/C}）
カラー：メイン {mainColor} ／ サブ {subColor}
キャッチコピー：「{heroTitle}」

よろしければ「OK」と送ってください。
```

### OKが来たら実行

**1. JSONを一時ファイルに書き出す（日本語の文字化け防止のため必須）**

```javascript
// スクラッチパスへ書き出す
{
  "name": "クラブ正式名称",
  "shortName": "SLUG",
  "mainColor": "#1a5c2d",
  "subColor": "#ffffff",
  "heroTitle": "キャッチコピー",
  "plan": "standard-02",
  "instagram": "https://www.instagram.com/...",
  "lineUrl": "https://line.me/...",
  "address": "住所",
  "phone": "電話番号",
  "email": "メールアドレス",
  "founded": 2010
}
```

**2. スクリプト実行**

```bash
node scripts/new-club.mjs "$(cat /path/to/temp.json)"
```

**3. 完了報告**

スクリプトの出力URLをそのままユーザーに伝える：

```
完成しました！

ホームページ：http://localhost:3000/deployments/{slug}/
Match Planner：http://localhost:3000/deployments/{slug}-match-planner/

写真の配置場所は PHOTO_GUIDE.md を確認してください：
deployments/{slug}/PHOTO_GUIDE.md
→ logo.png と hero.jpg を入れると一気に本物らしくなります！
```

サーバーが起動していない場合は先に起動する：

```bash
npx serve . --listen 3000
```

---

## 注意事項

- スラッグ（フォルダ名）は `shortName` を小文字・ハイフン区切りに変換して自動生成される
- 同名フォルダが既に存在する場合はエラーになる（重複防止）
- JSONBinは自動で新規作成される（既存クラブと完全に分離）
- APIキーは省略可能（デフォルトのマスターキーを使用）
- 質問UIダイアログ（AskUserQuestion）は使わず、テキストで直接質問する
